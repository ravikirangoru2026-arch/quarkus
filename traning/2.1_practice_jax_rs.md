# Implementing JAX-RS (RESTEasy Reactive) in Quarkus — Step by Step

This guide walks through building a REST API with Quarkus's RESTEasy Reactive (JAX-RS), writing unit/integration tests, and running the app — using a `User` resource as the working example.

---
Ref: https://quarkus.io

---

## Step 1: Generate the Project

```bash
mvn io.quarkus.platform:quarkus-maven-plugin:create \
    -DprojectGroupId=com.example \
    -DprojectArtifactId=user-service \
    -Dextensions="resteasy-reactive,resteasy-reactive-jackson,hibernate-orm-panache,jdbc-h2,rest-assured"

cd user-service
```

**Extensions used:**
| Extension | Purpose |
|---|---|
| `resteasy-reactive` | JAX-RS REST endpoint support |
| `resteasy-reactive-jackson` | JSON serialization/deserialization |
| `hibernate-orm-panache` | Simplified ORM for persistence |
| `jdbc-h2` | In-memory DB for local dev/tests |
| `rest-assured` | Test dependency for REST endpoint testing |

If adding to an existing project instead:
```bash
./mvnw quarkus:add-extension -Dextensions="resteasy-reactive,resteasy-reactive-jackson,hibernate-orm-panache,jdbc-h2,rest-assured"
```

## Step 3: Create the Entity

`src/main/java/com/example/User.java`
```java
package com.example;

import io.quarkus.hibernate.orm.panache.PanacheEntity;
import jakarta.persistence.Entity;
import jakarta.validation.constraints.Email;
import jakarta.validation.constraints.NotBlank;

@Entity
public class User extends PanacheEntity {

    @NotBlank
    public String name;

    @Email
    public String email;
}
```

---

## Step 4: Create the Service Layer

`src/main/java/com/example/UserService.java`
```java
package com.example;

import jakarta.enterprise.context.ApplicationScoped;
import jakarta.transaction.Transactional;
import java.util.List;

@ApplicationScoped
public class UserService {

    public List<User> findAll() {
        return User.listAll();
    }

    public User findById(Long id) {
        return User.findById(id);
    }

    @Transactional
    public User create(User user) {
        user.persist();
        return user;
    }

    @Transactional
    public boolean delete(Long id) {
        return User.deleteById(id);
    }
}
```

## Step 5: Create the JAX-RS Resource

`src/main/java/com/example/UserResource.java`
```java
package com.example;

import jakarta.inject.Inject;
import jakarta.validation.Valid;
import jakarta.ws.rs.*;
import jakarta.ws.rs.core.MediaType;
import jakarta.ws.rs.core.Response;
import java.util.List;

@Path("/users")
@Produces(MediaType.APPLICATION_JSON)
@Consumes(MediaType.APPLICATION_JSON)
public class UserResource {

    @Inject
    UserService userService;

    @GET
    public List<User> getAll() {
        return userService.findAll();
    }

    @GET
    @Path("/{id}")
    public Response getById(@PathParam("id") Long id) {
        User user = userService.findById(id);
        if (user == null) {
            return Response.status(Response.Status.NOT_FOUND).build();
        }
        return Response.ok(user).build();
    }

    @POST
    public Response create(@Valid User user) {
        User created = userService.create(user);
        return Response.status(Response.Status.CREATED).entity(created).build();
    }

    @DELETE
    @Path("/{id}")
    public Response delete(@PathParam("id") Long id) {
        boolean deleted = userService.delete(id);
        return deleted
            ? Response.noContent().build()
            : Response.status(Response.Status.NOT_FOUND).build();
    }
}
```

## Step 7: Write Unit Tests

Quarkus uses `@QuarkusTest` (JUnit 5) + REST Assured for endpoint-level testing. This boots the full Quarkus application context for the test.

`src/test/java/com/example/UserResourceTest.java`
```java
package com.example;

import io.quarkus.test.junit.QuarkusTest;
import io.restassured.http.ContentType;
import org.junit.jupiter.api.Test;

import static io.restassured.RestAssured.given;
import static org.hamcrest.Matchers.*;

@QuarkusTest
public class UserResourceTest {

    @Test
    public void testCreateAndGetUser() {
        // Create a user
        int id = given()
            .contentType(ContentType.JSON)
            .body("{\"name\":\"Alice\",\"email\":\"alice@example.com\"}")
            .when().post("/users")
            .then()
                .statusCode(201)
                .body("name", equalTo("Alice"))
                .body("email", equalTo("alice@example.com"))
                .extract().path("id");

        // Fetch the created user
        given()
            .when().get("/users/" + id)
            .then()
                .statusCode(200)
                .body("name", equalTo("Alice"));
    }

    @Test
    public void testGetAllUsers() {
        given()
            .when().get("/users")
            .then()
                .statusCode(200)
                .body("$", isA(java.util.List.class));
    }

    @Test
    public void testGetUserNotFound() {
        given()
            .when().get("/users/99999")
            .then()
                .statusCode(404);
    }

    @Test
    public void testCreateUserValidation() {
        // Missing required "name" field should fail Bean Validation
        given()
            .contentType(ContentType.JSON)
            .body("{\"email\":\"invalid\"}")
            .when().post("/users")
            .then()
                .statusCode(400);
    }

    @Test
    public void testDeleteUser() {
        int id = given()
            .contentType(ContentType.JSON)
            .body("{\"name\":\"ToDelete\",\"email\":\"del@example.com\"}")
            .when().post("/users")
            .then()
                .statusCode(201)
                .extract().path("id");

        given()
            .when().delete("/users/" + id)
            .then()
                .statusCode(204);

        given()
            .when().get("/users/" + id)
            .then()
                .statusCode(404);
    }
}
```

### Optional: Pure Unit Test for the Service Layer (without HTTP)

`src/test/java/com/example/UserServiceTest.java`
```java
package com.example;

import io.quarkus.test.junit.QuarkusTest;
import jakarta.inject.Inject;
import jakarta.transaction.Transactional;
import org.junit.jupiter.api.Test;

import static org.junit.jupiter.api.Assertions.*;

@QuarkusTest
public class UserServiceTest {

    @Inject
    UserService userService;

    @Test
    @Transactional
    public void testCreateUser() {
        User user = new User();
        user.name = "Bob";
        user.email = "bob@example.com";

        User created = userService.create(user);

        assertNotNull(created.id);
        assertEquals("Bob", created.name);
    }

    @Test
    @Transactional
    public void testFindByIdReturnsNullWhenMissing() {
        assertNull(userService.findById(123456L));
    }
}
```

## Step 8: Run the Tests

```bash
./mvnw test
```

Run a single test class:
```bash
./mvnw test -Dtest=UserResourceTest
```

Continuous testing (in dev mode, press `r` to re-run on save):
```bash
./mvnw quarkus:dev
```

Integration tests against the packaged artifact (`@QuarkusIntegrationTest`, useful for native build verification):
```bash
./mvnw verify
```

```bash
./mvnw clean package -Pnative -Dquarkus.native.container-build=true

```

## Summary Flow

```
Generate project → Configure datasource → Define entity
       → Build service layer → Build JAX-RS resource
       → Run in dev mode → Write @QuarkusTest cases
       → Run tests (mvnw test) → Package & run (mvnw package / java -jar)
```

| Layer | Annotation/Tool Used |
|---|---|
| Entity | `@Entity`, Panache `PanacheEntity` |
| Service | `@ApplicationScoped`, `@Transactional` |
| REST Resource | `@Path`, `@GET`/`@POST`/`@DELETE`, `@Produces`/`@Consumes` |
| Testing | `@QuarkusTest`, REST Assured (`given()/when()/then()`) |
| Validation | `@Valid`, `@NotBlank`, `@Email` |
