Covers JAX-RS (RESTEasy Reactive / Quarkus REST) annotations commonly used to build REST APIs in Quarkus.

## 1. Class & Resource-Level Annotations

### `@Path`
**Why:** Maps a class or method to a URI path.
```java
@Path("/users")
public class UserResource { }
```

### `@ApplicationPath`
**Why:** Defines the base URI for all JAX-RS resources in the app (set on the Application subclass).
```java
@ApplicationPath("/api")
public class RestApplication extends Application { }
```

## 2. HTTP Method Annotations

### `@GET`
**Why:** Marks a method to handle HTTP GET requests (retrieve data).
```java
@GET
@Path("/{id}")
public User getUser(@PathParam("id") Long id) {
    return User.findById(id);
}
```

### `@POST`
**Why:** Marks a method to handle HTTP POST requests (create a resource).
```java
@POST
@Transactional
public User createUser(User user) {
    user.persist();
    return user;
}
```

### `@PUT`
**Why:** Marks a method to handle HTTP PUT requests (full update of a resource).
```java
@PUT
@Path("/{id}")
@Transactional
public User updateUser(@PathParam("id") Long id, User updated) {
    User user = User.findById(id);
    user.name = updated.name;
    return user;
}
```

### `@PATCH`
**Why:** Marks a method to handle HTTP PATCH requests (partial update of a resource).
```java
@PATCH
@Path("/{id}")
@Transactional
public User patchUser(@PathParam("id") Long id, Map<String, Object> fields) {
    User user = User.findById(id);
    fields.forEach((k, v) -> applyField(user, k, v));
    return user;
}
```

### `@DELETE`
**Why:** Marks a method to handle HTTP DELETE requests (remove a resource).
```java
@DELETE
@Path("/{id}")
@Transactional
public void deleteUser(@PathParam("id") Long id) {
    User.deleteById(id);
}
```

### `@HEAD`
**Why:** Handles HTTP HEAD requests â€” returns headers only, no body (useful for existence checks).
```java
@HEAD
@Path("/{id}")
public Response checkUser(@PathParam("id") Long id) {
    return User.findByIdOptional(id).isPresent()
        ? Response.ok().build() : Response.status(404).build();
}
```

### `@OPTIONS`
**Why:** Handles HTTP OPTIONS requests â€” typically used to describe allowed methods (CORS preflight).
```java
@OPTIONS
public Response options() {
    return Response.ok()
        .header("Allow", "GET,POST,PUT,DELETE")
        .build();
}
```

## 3. Parameter Binding Annotations

### `@PathParam`
**Why:** Binds a URI path segment to a method parameter.
```java
@GET
@Path("/{id}")
public User get(@PathParam("id") Long id) { ... }
```

### `@QueryParam`
**Why:** Binds a URL query string parameter to a method parameter.
```java
@GET
public List<User> search(@QueryParam("name") String name) {
    return User.list("name", name);
}
```

### `@HeaderParam`
**Why:** Binds an HTTP request header value to a method parameter.
```java
@GET
public String getAgent(@HeaderParam("User-Agent") String agent) {
    return agent;
}
```

### `@CookieParam`
**Why:** Binds a cookie value from the request to a method parameter.
```java
@GET
public String getSession(@CookieParam("session-id") String sessionId) {
    return sessionId;
}
```

### `@FormParam`
**Why:** Binds a form field (application/x-www-form-urlencoded) to a method parameter.
```java
@POST
@Consumes(MediaType.APPLICATION_FORM_URLENCODED)
public Response login(@FormParam("username") String username) {
    return Response.ok().build();
}
```

### `@MatrixParam`
**Why:** Binds matrix-style URI parameters (e.g. `/users;sort=asc`) to a method parameter.
```java
@GET
public List<User> list(@MatrixParam("sort") String sort) { ... }
```

### `@BeanParam`
**Why:** Aggregates multiple parameter annotations into a single bean object.
```java
@GET
public List<User> filter(@BeanParam UserFilter filter) { ... }
```

### `@DefaultValue`
**Why:** Supplies a fallback value when the bound parameter is absent.
```java
@GET
public List<User> list(@QueryParam("limit") @DefaultValue("10") int limit) { ... }
```

### `@RestPath` / `@RestQuery` / `@RestHeader` / `@RestForm` *(Quarkus REST â€” formerly RESTEasy Reactive shortcuts)*
**Why:** Quarkus-specific shorthand annotations that infer the parameter name from the method parameter name, reducing boilerplate.
```java
@GET
@Path("/{id}")
public User get(@RestPath Long id) { ... }
```

## 4. Content Negotiation Annotations

### `@Produces`
**Why:** Declares the MIME type(s) a resource method returns.
```java
@GET
@Produces(MediaType.APPLICATION_JSON)
public List<User> getUsers() { ... }
```

### `@Consumes`
**Why:** Declares the MIME type(s) a resource method accepts as input.
```java
@POST
@Consumes(MediaType.APPLICATION_JSON)
public User create(User user) { ... }
```

---

## 5. Context & Injection Annotations

### `@Context`
**Why:** Injects JAX-RS context objects (UriInfo, HttpHeaders, Request, SecurityContext) into a resource.
```java
@GET
public String getUri(@Context UriInfo uriInfo) {
    return uriInfo.getPath();
}
```

### `@Suspended` (with `AsyncResponse`)
**Why:** Marks a parameter for asynchronous request processing.
```java
@GET
public void asyncGet(@Suspended AsyncResponse response) {
    response.resume("done");
}
```

## 6. Security Annotations

### `@RolesAllowed`
**Why:** Restricts access to a method/class to users with specific roles.
```java
@RolesAllowed("admin")
@DELETE
@Path("/{id}")
public void delete(@PathParam("id") Long id) { ... }
```

### `@PermitAll`
**Why:** Allows unrestricted access to a method/class (overrides class-level restrictions).
```java
@PermitAll
@GET
public List<User> publicList() { ... }
```

### `@DenyAll`
**Why:** Denies all access to a method, regardless of role.
```java
@DenyAll
@DELETE
public void disabledEndpoint() { ... }
```

### `@Authenticated`
**Why:** Requires the caller to be authenticated (any valid identity), without role restriction.
```java
@Authenticated
@GET
@Path("/profile")
public User getProfile(@Context SecurityContext ctx) { ... }
```

## 7. Validation Annotations (Bean Validation, used in REST payloads)

### `@Valid`
**Why:** Triggers Bean Validation on the annotated request body/parameter.
```java
@POST
public User create(@Valid User user) { ... }
```

### `@NotNull`, `@NotBlank`, `@Size`, `@Email`, etc.
**Why:** Declares field-level validation constraints on a DTO/entity used in REST requests.
```java
public class User {
    @NotBlank
    public String name;

    @Email
    public String email;
}
```

## 8. CDI / Lifecycle Annotations Commonly Paired with REST Resources

### `@ApplicationScoped`
**Why:** Defines the resource bean as a singleton-scoped CDI bean (default for JAX-RS resources in Quarkus).
```java
@ApplicationScoped
@Path("/users")
public class UserResource { ... }
```

### `@Inject`
**Why:** Injects a CDI bean (e.g., a service or repository) into the REST resource.
```java
@Path("/users")
public class UserResource {
    @Inject
    UserService userService;
}
```

### `@Transactional`
**Why:** Wraps the method execution in a JTA transaction â€” required for write operations using Hibernate ORM/Panache.
```java
@POST
@Transactional
public User create(User user) {
    user.persist();
    return user;
}
```

## 9. Documentation Annotations (SmallRye OpenAPI)

### `@Tag`
**Why:** Groups endpoints under a named category in the generated OpenAPI/Swagger documentation.
```java
@Tag(name = "Users", description = "User management operations")
@Path("/users")
public class UserResource { ... }
```

### `@Operation`
**Why:** Provides a summary/description for a specific REST operation in the API docs.
```java
@Operation(summary = "Get user by ID")
@GET
@Path("/{id}")
public User get(@PathParam("id") Long id) { ... }
```

### `@APIResponse`
**Why:** Documents possible HTTP response codes and payloads for an endpoint.
```java
@APIResponse(responseCode = "404", description = "User not found")
@GET
@Path("/{id}")
public User get(@PathParam("id") Long id) { ... }
```

### `@Parameter`
**Why:** Documents an individual method parameter (name, description, example) in the OpenAPI spec.
```java
@Parameter(name = "id", description = "User ID", example = "123")
@GET
@Path("/{id}")
public User get(@PathParam("id") Long id) { ... }
```

## Quick Reference Table

| Annotation | Category | Purpose |
|---|---|---|
| `@Path` | Routing | Maps class/method to a URI |
| `@GET` / `@POST` / `@PUT` / `@PATCH` / `@DELETE` / `@HEAD` / `@OPTIONS` | HTTP Method | Maps method to HTTP verb |
| `@PathParam` | Binding | Bind URI path segment |
| `@QueryParam` | Binding | Bind query string param |
| `@HeaderParam` | Binding | Bind HTTP header value |
| `@CookieParam` | Binding | Bind cookie value |
| `@FormParam` | Binding | Bind form field |
| `@BeanParam` | Binding | Aggregate params into a bean |
| `@DefaultValue` | Binding | Fallback parameter value |
| `@RestPath`/`@RestQuery`/etc. | Binding (Quarkus shorthand) | Infer param name automatically |
| `@Produces` | Content negotiation | Response MIME type |
| `@Consumes` | Content negotiation | Accepted request MIME type |
| `@Context` | Injection | Inject JAX-RS context objects |
| `@Suspended` | Async | Async response handling |
| `@RolesAllowed` | Security | Role-based access control |
| `@PermitAll` / `@DenyAll` | Security | Allow/deny all access |
| `@Authenticated` | Security | Require valid identity |
| `@Valid` | Validation | Trigger bean validation |
| `@ApplicationScoped` | CDI | Resource bean scope |
| `@Inject` | CDI | Dependency injection |
| `@Transactional` | Persistence | Wrap method in a transaction |
| `@Tag` / `@Operation` / `@APIResponse` / `@Parameter` | OpenAPI | API documentation metadata |
