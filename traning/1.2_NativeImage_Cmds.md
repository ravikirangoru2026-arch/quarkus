# Quarkus Native Image

## 🔍 What is Quarkus Native Image?
- A **[Quarkus Native Image](ca://s?q=Quarkus_native_image)** is a compiled executable of a Quarkus application using **[GraalVM](ca://s?q=Quarkus_GraalVM_native_support)**.  
- Instead of running on the JVM, the application is compiled **ahead-of-time (AOT)** into a **self-contained binary**.  
- This binary includes only the required classes and dependencies, resulting in **fast startup** and **low memory usage**.

# Quarkus — Packaging, Build, and Run Commands

## 1. Project Creation

```bash
mvn io.quarkus.platform:quarkus-maven-plugin:create \
    -DprojectGroupId=com.example \
    -DprojectArtifactId=my-app \
    -Dextensions="resteasy-reactive,resteasy-reactive-jackson"
```

Gradle equivalent:
```bash
gradle init -DprojectGroupId=com.example -DprojectArtifactId=my-app
```

---

## 2. Dev Mode (Live Coding)

```bash
./mvnw quarkus:dev
```

Gradle:
```bash
./gradlew quarkusDev
```

- Hot reload on code/resource changes
- Continuous testing: press `r` in dev console
- Custom debug port:
```bash
./mvnw quarkus:dev -Ddebug=5006
```

---

## 3. Build (JVM Mode — Standard Jar)

```bash
./mvnw clean package
```

Output: `target/quarkus-app/quarkus-run.jar` (fast-jar, default since Quarkus 1.12+)

Run the built jar:
```bash
java -jar target/quarkus-app/quarkus-run.jar
```

Skip tests during build:
```bash
./mvnw clean package -DskipTests
```

### Uber-Jar (Single Fat Jar)

```bash
./mvnw clean package -Dquarkus.package.jar.type=uber-jar
```

Run:
```bash
java -jar target/my-app-1.0.0-SNAPSHOT-runner.jar
```

---

## 4. Native Build (GraalVM)

### Using locally installed GraalVM
```bash
./mvnw clean package -Pnative
```

### Using container build (no local GraalVM required — recommended)
```bash
./mvnw clean package -Pnative -Dquarkus.native.container-build=true
```

Run the native executable:
```bash
./target/my-app-1.0.0-SNAPSHOT-runner
```

### Native build with custom builder image
```bash
./mvnw clean package -Pnative \
    -Dquarkus.native.container-build=true \
    -Dquarkus.native.builder-image=quay.io/quarkus/ubi-quarkus-native-image:22.3-java17
```

---

## 5. Container Image Build

### Jib (no Dockerfile needed)
```bash
./mvnw clean package -Dquarkus.container-image.build=true \
    -Dquarkus.container-image.builder=jib
```

### Docker
```bash
./mvnw clean package -Dquarkus.container-image.build=true \
    -Dquarkus.container-image.builder=docker
```

### Build native + container image in one step
```bash
./mvnw clean package -Pnative \
    -Dquarkus.native.container-build=true \
    -Dquarkus.container-image.build=true
```

### Push image to registry
```bash
./mvnw clean package -Dquarkus.container-image.build=true \
    -Dquarkus.container-image.push=true \
    -Dquarkus.container-image.registry=docker.io \
    -Dquarkus.container-image.group=myorg \
    -Dquarkus.container-image.name=my-app \
    -Dquarkus.container-image.tag=1.0.0
```

---

## 6. Running Pre-built Artifacts

### JVM mode (fast-jar)
```bash
java -jar target/quarkus-app/quarkus-run.jar
```

### With custom profile and port
```bash
java -Dquarkus.profile=prod -Dquarkus.http.port=8081 \
    -jar target/quarkus-app/quarkus-run.jar
```

### Native executable
```bash
./target/my-app-1.0.0-SNAPSHOT-runner -Dquarkus.http.port=8081
```

### Docker run (after container image build)
```bash
docker run -i --rm -p 8080:8080 myorg/my-app:1.0.0
```

## 7. Testing

```bash
./mvnw test
```

Integration tests (against packaged app):
```bash
./mvnw verify
```

Continuous testing in dev mode:
```bash
./mvnw quarkus:dev
# then press 'r' in the terminal
```

## 8. Useful Build Flags Reference

| Flag | Purpose |
|---|---|
| `-Pnative` | Activates native build profile |
| `-Dquarkus.native.container-build=true` | Builds native image inside container (no local GraalVM needed) |
| `-Dquarkus.package.jar.type=uber-jar` | Produces a single fat jar |
| `-Dquarkus.container-image.build=true` | Builds container image after packaging |
| `-Dquarkus.container-image.push=true` | Pushes built image to registry |
| `-DskipTests` | Skips test execution |
| `-Dquarkus.profile=<profile>` | Activates a specific config profile (dev/test/prod) |
| `-Ddebug=<port>` | Sets remote debug port in dev mode |
